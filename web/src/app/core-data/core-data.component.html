Tips
An event represents a collection of one or more sensor readings. Some sensors or devices are only providing a single value – a single reading - at a time. Other sensors spew multiple values whenever they are read. An event must have at least one reading. Events are associated to a sensor or device.

The following data is only for the convenience of the user to view the data format of the data center, which does not make any sense to manipulate!
The dynamic display effect of the data is that only the API of the Core Data service is called periodically, with intervals of 3 seconds.
